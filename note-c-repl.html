C REPL

This note is about how to setup C read print evaluate loop (REPL) by using Emacs, GDB and Make.

Refer to this for the relevant setup in Emacs and a typical Makefile


Steps to setup:

  1. Open the top level source file in Emacs (let's called it main.c and should have a main function)
  2. Run M-x gud-gdb
  3. When prompted enter: gud out
  4. GDB should started in a buffer named *gud-out*
  5. In the main.c buffer run C-c C-c to make; check the output in GDB buffer
  6. If it asks you to use the "file" command then in the GDB prompt enter: file out
  7. In the main.c buffer run C-c C-c to make again; check the output in GDB buffer

Now the REPL is ready:

  • You can compile the changes to the source codes by running C-c C-c in the main.c buffer
  • then you can evaluate code (function call) in main.c buffer and print result to the GDB buffer by using inline comment and run C-<return> while the cursor is on that line; for examples:
    // 8 + 8
    // size(int)
    // sq(8)
